**Механизм:** условный `UPDATE WHERE status = 'assigned'` в транзакции + WAL-режим SQLite.
Только один из двух параллельных запросов получает `rowCount() = 1`. Второй видит 0 и получает 409.

## Выявленные и исправленные баги

| Баг | Причина | Исправление |
|-----|---------|-------------|
| `http_response_code(409)` сбрасывался в 302 | PHP автоматически ставит 302 при `header('Location:')` | На 409-ответ рендерим страницу напрямую без редиректа |
| Листинг директории на `http://127.0.0.1/test8/` | `.htaccess` содержал `RewriteCond !-d`, исключавшее директории | Убрано условие `!-d`; правило охватывает все запросы кроме файлов |
| SQLite "attempt to write a readonly database" | Apache (`www-data`) не имел прав записи | `chmod 666 database.sqlite`, `chmod o+w database/` |
